# BKI-missing-applications-finder
 # Автоматическая сверка заявок и договоров БКИ для поиска расхождений UUID.

## 1.	Проблема и решение
Проблема: Специалисты в сфере БКИ вынуждены вручную сопоставлять огромные XML-файлы, чтобы найти расхождение между UUID заявок и договоров.

Решение: Этот скрипт полностью автоматизирует рутину. Он мгновенно сопоставляет базу договоров с поданными заявками и выводит только список UUID договоров, к которым отсутствует заявка.

## 2.	Алгоритм работы и оптимизация 
Вместо использования медленных вложенных циклов, которые могли бы зависать  при проверке файлов на сотни тысяч строк, я решила использовать множества, которые убирают дубликаты UUID и эффективно находят расхождения между ними.
Скрипт считывает файлы с помощью встроенной библиотеки xml.etree.ElementTree,
При сборе UUID программа автоматически убирает лишние пробелы методом .strip() и приводит строки к нижнему регистру через метод .lower()
Скрипт находит разницу между множеством договоров и множеством заявок с помощью математической операции разности множеств (-).

## 3.	Как использовать скрипт с вашими файлами.
Для работы скрипта необходимо подготовить файлы отчетности:
Положите оба XML-файла в ту же папку, где находится скрипт. Файл с заявками нужно переименовать в applications.xml, а файл с договорами – в contracts.xml. 
При желании оригинальные названия файлов можно изменить в коде – в строках, где происходит загрузка: 
tree_application(ЕСЛИ ДОГОВОР – tree_contracts) = ET.parse('ОРИГИНАЛЬНОЕ НАЗВАНИЕ ФАЙЛА')


## 4.	Настройка пространства имен (XML Namespace) и тегов.
Поскольку файлы БКИ используют индивидуальные XML-схемы, перед запуском нужно проверить переменную ⁠xml_ns⁠: 
 Переменная ⁠xml_ns⁠ должна содержать адрес вашей схемы внутри фигурных скобок ⁠{}⁠ (его можно подсмотреть, если открыть ваш XML-файл через Блокнот и найти в начале строку ⁠xmlns="..."⁠).
Скрипт ориентирован на стандартные теги: ⁠<InfoPart>⁠ для заявок и ⁠<Trade>⁠ для договоров. Если в ваших структурах теги называются иначе, их можно легко заменить в циклах ⁠for⁠.

## 5. Тестовые файлы 
В репозитории подготовлены тестовые файлы для проверки работы скрипта (все данные фейковые, XML-файл имеет такую же структуру, как и у реального файла):
* `applications.xml` - пример файла с заявками.
* `contracts.xml` - пример файла с договорами, в котором намеренно допущены расхождения (есть договоры без соответствующих заявок), чтобы вы могли сразу протестировать скрипт в действии.

## 6.	Исходный код.
```python
import xml.etree.ElementTree as ET

#загрузка и чтение XML-файлов (заявки)
tree_application = ET.parse('applications.xml')
root_application = tree_application.getroot()

#загрузка и чтение XML-файлов (договоры)
tree_contract = ET.parse('contracts.xml')
root_contract = tree_contract.getroot()

#Работа с Namespaces
xml_ns = '{http://cbr.ru}'  #(Сюда вставляется актуальный xmlns из рабочих файлов)

uuid_application = set()
uuid_contract = set()
invalid_contracts = set()

#Сбор UUID из файла заявок
for application in root_application.iter(f'{xml_ns}InfoPart'):
    uuid_tag_a = application.find(f'{xml_ns}Uuid')
    uuid_application.add(uuid_tag_a.text.strip().lower())
#Сбор UUID из файла договоров
for contract in root_contract.iter(f'{xml_ns}Trade'):
    uuid_tag_c = contract.find(f'{xml_ns}Uuid')
    uuid_contract.add(uuid_tag_c.text.strip().lower())

#Поиск договоров, для которых не были найдены заявки
invalid_contracts = uuid_contract - uuid_application

#Вывод договоров без заявок
print(f'UUID договоров, к которым отсутствует заявка: {invalid_contracts}')
```
